# План продвижения Memora Solutions

Дата: 28 мая 2026 • Аудитория: рунет (приоритет — СПб), частично EN для десктоп-продукта.

---

## 1. Позиционирование

Memora Solutions — это не «агентство», а **витрина живых пет-продуктов одного разработчика-архитектора**. Сила бренда — в том, что всё работает прямо сейчас и приносит пользу: бот напоминает о ДР, радар сам тащит горящие туры, Pomodoro можно скачать и запустить.

Главный посыл во всех каналах: **«Я делаю инструменты, которыми пользуюсь сам, и отдаю их людям».** Это честнее и убедительнее, чем «инновационные решения для бизнеса».

Каждый продукт продвигается **отдельно под свою аудиторию**, а сайт-портфолио — это точка сборки и доказательство компетенции (особенно раздел «Внутренние разработки» / Product Dev OS).

## 3. Площадки по приоритету (ROI / усилие)

**Tier 1 — делать в первую очередь:**
1. **Свой Telegram-канал** — лента апдейтов по всем продуктам. Дёшево, твой, греет лояльную аудиторию. Анонс каждой фичи + закулисье разработки.
2. **VK** — основная масса рускоязычной нетехнической аудитории (Travel Radar, Bday-bot). Постинг в тематические сообщества + своя группа.
3. **Habr** — для Pomodoro, Kanban и **статьи про Product Dev OS / как я в одиночку веду 6 продуктов**. Habr даёт долгий органический трафик и репутацию инженера.

**Tier 2 — добавить после Tier 1:**
4. **vc.ru / DTF** — формат «как я сделал X за выходные», истории продукта, метрики. Хорошо для Pomodoro и кейса портфолио.
5. **Pikabu** — Bday-bot и Travel Radar заходят как «полезная штука», вирусится при удачной подаче.
6. **Product Hunt (EN)** — только для Pomodoro как скачиваемого приложения. Нужен англоязычный лендинг и GitHub Releases с готовым билдом.

**Tier 3 — точечно:**
7. **Reddit** (r/SideProject, r/productivity, r/pomodoro) — для Pomodoro, на английском.
8. **Профильные TG-чаты по TON** — для Wallet.
9. **GitHub** — оформить репозитории, README с гифками, топики; органический трафик от разработчиков.

## 4. Что починить до запуска промо (блокеры)

- [ ] **Pomodoro: реальный билд в GitHub Releases.** Сейчас кнопка «Скачать» ведёт на пустую страницу релизов — это убьёт первый же поток трафика. Сначала собрать и выложить `.exe`/`.dmg`, потом анонсить.
- [ ] **Брендовый логотип Pomodoro (томат)** вместо временной картинки — нужен для Product Hunt/Reddit, там визуал решает.
- [ ] **Англоязычная версия лендинга Pomodoro** — для EN-площадок (PH, Reddit).
- [ ] **OG-теги и превью-картинки** для всех страниц — чтобы ссылки в соцсетях выглядели прилично. Проверить, что есть `og:image`, `og:title`, `description`.
- [ ] **Аналитика** — поставить хотя бы Plausible/Yandex.Metrica, иначе промо вслепую.

## 6. Тон и формат

- **Нетехнические площадки (VK, Pikabu):** язык пользы, без жаргона. «Открыл — увидел туры — нажал — улетел».
- **Технические (Habr, Reddit, vc.ru):** честный разбор решений и компромиссов, код/гифки/метрики. Аудитория ценит «как», а не «вау».
- **Везде:** показывать продукт в действии (гифка/скрин) в первых 3 секундах, ссылку — рано и заметно.

---

## 7. Метрики успеха

- TG-канал: подписчики и охват поста (цель — стабильный рост, не разовый всплеск).
- Сайт: уникальные визиты, источники, глубина (доходят ли до /pomodoro и /internal).
- Pomodoro: скачивания из Releases, апвоуты на PH/Reddit.
- Travel Radar: возвраты (retention) — главный сигнал, что лента реально полезна.
- Лиды/контакты с раздела «Создатель» — конверсия портфолио в работу/заказы.
