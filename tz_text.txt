1. Позиционирование проекта
Мемора Солюшнс — проект, выросший из тишины, наблюдения и желания быть полезным в самом главном.
Мы разрабатываем цифровые инструменты, чтобы помочь вам:
Улучшать качество отношений
Планировать (очень недооцененный навык)
Освободить время для счастья
Этот проект — не про быструю выгоду. Он о долгом пути и настоящих переменах. Каждый инструмент мы создаем сами, с нуля: от идеи до реализации. С вниманием к деталям, опыту и потребностям людей.

6. Мемора Радар Путешествий — Техническое задание
6.1. Концепция
Мемора Радар Путешествий — персональная система мониторинга выгодных путешествий. Фаза 1 — горящие пакетные туры из Санкт-Петербурга. Далее — агрегация нескольких каналов экономии: горящие туры, авиабилеты, error fares, long-stay скидки, бонусные мили.
Telegram-бот: @MemoraTravelRadarBot
6.2. Архитектура данных (закладывается на старте)
Проектировать универсально, даже если первая версия работает с одним источником.

6.3. Фазы реализации
Фаза 1 — Интерфейс + горящие пакетные туры
Срок: 2–3 недели. Приоритет: максимальный.
Шаг 1 — Веб-интерфейс:
Страница Мемора Радар Путешествий на сайте
Форма поиска: город вылета, направление (Таиланд, Вьетнам, Турция, Китай/Хайнань, Бали), даты, звёзды, бюджет, ночи
Лента предложений (Deal) с карточками: направление, отель, звёзды, ночи, питание, оператор, цена, скидка, дата вылета
Фильтры и сортировка
Блок "Полезные сервисы" (п. 6.5) — ценность с первого дня
Адаптив: мобильная и десктопная версия

Шаг 2 — Подключение данных:
Sletat.ru JSON API: регистрация + тестовая лицензия (support@sletat.ru)
Backend: CreateSearch → GetSearchState → GetTours
Cron: автопоиск по 5 направлениям каждые 2 часа
Telegram-бот @MemoraTravelRadarBot: уведомления при цене ниже порога
Сохранение истории цен
Фаза 2 — Расширение источников
Срок: через 1–2 месяца после Фазы 1.
Мониторинг авиабилетов (Aviasales API / парсинг) — для Бали и нестандартных маршрутов
Парсинг Telegram-каналов с горящими турами (Telethon/Pyrogram)
Сравнение: пакет vs самостоятельная сборка
Графики динамики цен по направлениям
Фаза 3 — Travel Intelligence
Срок: по мере развития и обратной связи пользователей.
Error fares — мониторинг ошибочных тарифов авиакомпаний
Калькулятор бонусных миль (Аэрофлот Бонус, S7 Priority, Тинькофф Travel)
Long-stay скидки — месячные тарифы Airbnb/Booking
Ценовой арбитраж — бронирование через разные точки продажи
House sitting — интеграция с платформами (TrustedHousesitters, HouseCarers)
AI-рекомендации — персонализированные предложения

6.4. Sletat.ru API — техническая справка

6.5. Блок "Полезные сервисы" (публикуется на старте)
Курируемый список внешних ресурсов. Каждый сервис: название, краткое описание, раскрывающийся аккордеон "Как пользоваться" (3–5 шагов). Администратор управляет списком через панель.
Категория A — Агрегаторы пакетных туров

Категория B — Авиабилеты и ошибочные тарифы

Категория C — Отели и проживание

Категория D — Бонусные мили и кэшбэк

Категория E — AI-планировщики и инструменты

Категория F — Лайфхаки и арбитраж

UX-реализация: сетка карточек по категориям. Каждая карточка: название, одна строка описания, аккордеон "Как пользоваться" (3–5 шагов). Ссылки в новой вкладке. Администратор управляет через панель.

6.6. Интерфейс — ключевые экраны
Главный экран:  форма поиска + лента горящих предложений. Фильтры. Стиль наследуется от сайта (Obsidian Minimal).
Карточка Deal:  флаг + направление, курорт, звёзды, ночи, питание, оператор, цена (текущая + зачёркнутая + % скидки), дата вылета, метка срочности. Ultra-hot метка при скидке >50%.
Полезные сервисы:  сетка карточек по категориям A–F. Под основной лентой или рядом.
Загрузка:  скелетоны карточек + "Сканируем предложения..." (Sletat API отвечает 1 сек — 2 мин).
Пустое состояние:  "Туров не найдено. Измените параметры или посмотрите полезные сервисы ниже."

6.7. Telegram-бот @MemoraTravelRadarBot
Пользователь задаёт фильтры: направления, макс. бюджет, мин. звёзды, ночи, окно вылета
Мгновенное сообщение при нахождении подходящего предложения + ссылка на бронирование
Команды: /start, /settings, /status, /pause, /resume
Частота проверки: каждые 2 часа (настраивается)

7. UI Style Specification
Референс: Obsidian Minimal Theme
Принципы: минимализм, много воздуха, мягкие контрасты, спокойные акценты, минимум цветов одновременно.
Правило: 90% интерфейса — нейтральный, 10% — фирменные цвета.
7.1. Цветовая палитра
Primary Accent — Gold (цвет логотипа)
Кнопки, ссылки, активные элементы, выделение.

Growth Green (левая часть логотипа)
Статусы, подсветка карточек, теги, вторичные акценты.

Clarity Blue (правая часть логотипа)
Информационные блоки, secondary buttons, hover-эффекты.

Нейтральные цвета

7.2. Распределение цвета
Нейтральный интерфейс: 85–90%. Gold: 5–7%. Green: 2–4%. Blue: 2–4%. Цвета не должны конкурировать.
7.3. Формы и размеры
Border radius: 12px | Card radius: 16px | Buttons: 10px
Карточки: background #FFFFFF, border 1px solid #E6E3DD
Тени: box-shadow 0 6px 16px rgba(0,0,0,0.05). Hover: 0 8px 20px rgba(0,0,0,0.08)
7.4. Кнопки
Primary: background #C89B6D, text #FFFFFF, radius 10px. Hover: #D6AE82
Secondary: background transparent, border 1px solid #E6E3DD, text #2B2B2B. Hover: border #C89B6D
7.5. Типографика
Шрифты: Inter (основной), Manrope (альтернативный)
H1 — 32px, H2 — 24px, H3 — 20px, Body — 16px, Small — 14px
7.6. Иконки
Стиль outline, толщина линии 1.5–2px. Библиотеки: Lucide, Heroicons, Phosphor.
7.7. Анимации
Очень лёгкие. transition: 0.18s ease. Hover: изменение цвета + увеличение тени.
7.8. Тёмная тема
Инвертировать нейтральные цвета. Акцентные (Gold, Green, Blue) — сохранить. Background: #1A1A1E, Surface: #242428, Text Primary: #E5E5E5, Border: #3A3A3E.
